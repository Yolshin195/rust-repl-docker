# Rust REPL in Docker/Podman

Этот проект предоставляет готовый к использованию образ Docker/Podman с предустановленным [evcxr_repl](https://github.com/evcxr/evcxr). Он идеально подходит для экспериментов с кодом на Rust, быстрого прототипирования или обучения, не засоряя вашу основную операционную систему установкой различных версий Rust и системных библиотек.

## Зачем это нужно?

* **Изоляция:** Все зависимости Rust и скомпилированные файлы остаются внутри контейнера.
* **Чистота системы:** Вам не нужно устанавливать `rustup` или системные пакеты вроде `libssl-dev` в свою основную ОС.
* **Портативность:** Один и тот же REPL будет работать одинаково на любом компьютере, где установлены Docker или Podman.
* **Удобство:** Простой `Makefile` автоматизирует процесс сборки и запуска.

## Предварительные требования

* [Docker](https://www.docker.com/) или [Podman](https://podman.io/).
* `make` (установлен по умолчанию в большинстве систем Linux/macOS).

## Использование

### 1. Клонирование репозитория

```bash
git clone git@github.com:Yolshin195/rust-repl-docker.git
cd rust-repl-docker

```


### 2. Сборка образа

Чтобы собрать образ с установленным `evcxr`, выполните:

```bash
make build

```


### 3. Запуск REPL

Запустите интерактивную оболочку Rust:

```bash
make run

```

Теперь вы можете писать код на Rust прямо в терминале. Выход из REPL: `:quit` или `Ctrl+D`.

### 4. Очистка

Чтобы удалить созданный образ, выполните:

```bash
make clean

```

## Как это работает

* **Dockerfile:** Используется официальный образ `rust:1.96-slim-bookworm`. Мы устанавливаем системные зависимости (`pkg-config`, `libssl-dev`), создаем непривилегированного пользователя для безопасности и устанавливаем `evcxr_repl`.
* **Makefile:** Автоматически определяет, используете ли вы `docker` или `podman`, что делает сборку максимально простой.
